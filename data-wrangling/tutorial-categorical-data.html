Tidy data

Все счастливые семьи похожи друг на друга, каждая несчастливая семья несчастлива по-своему.

As Tolstoy write in Anna Karenina, “all happy families are alike; each unhappy family is unhappy in its own way.” The same is true of tidy data. All tidy datasets are alike; each untidy dataset is untidy in its own way. The three characteristics of tidy datasets are as follows:

  • Each variable forms a column,
  • Each observation forms a row, and
  • Each type of observational unit forms a table.

Today’s dataset

Today’s dataset was posted to Kaggle by Mario Tormo Romero, who scraped the data from the Pokemon Database and Serebii. It has been slightly modified for the purposes of this tutorial class.

The dataset contains information on all Pokemon, including alternate forms, through Generation 8 (Pokemon Sword and Shield). Pokemon have various characteristics unique to each one. The variables in the pokemon dataset are listed below.

  • pokedex_number: a catelogue number of each Pokemon
  • generation: which video game generation the Pokemon is from
  • leg_status: whether a Pokemon is sub-legendary, legendary, mythical, or none
  • type_1, type_2: what “type” a Pokemon is (fire, water, electric, etc.)
  • bst: the base stat total of a Pokemon (a proxy for its effectiveness in combat)
  • hp, atk, def, spa, spd, spe: a Pokemon’s battle statistics (HP, attack, defense, special attack, special defense, and speed, respectively)

Factors in R

R uses factors to handle categorical variables: variables that have a fixed set of possible values. Internally, factor variables are stored as integers that correspond to the character values you see when the factor is used. To cast a variable as a factor variable, you can use the as.factor() function that is a part of base R. Take a look at the data types for the variables in the following example, before and after we cast generation, leg_status, type_1, and type_2 as factors:

pokemon %>% 
  select(name, generation, type_1) %>% 
  slice(1:3)

vs.

pokemon_cat <- pokemon %>% 
  mutate(generation = as.factor(generation),
         leg_status = as.factor(leg_status),
         type_1 = as.factor(type_1),
         type_2 = as.factor(type_2))

pokemon_cat %>% 
  select(name, generation, type_1) %>% 
  slice(1:3)

The tools for manipulating factors in base R are often unwieldy to use, and it is sometimes difficult to perform even basic factor-related tasks. Luckily, we can use the forcats package, a suite of tools in the tidyverse that perform common data wrangling tasks with factors in a tidy-data-conscious way.

Abilities include:

  • Ordering by frequency
  • Reordering factor levels
  • Recoding factors
  • Collapsing levels
  • Lumping categories together
  • …and more!

Diving into forcats

Let’s begin by visualizing legendary status. Run the following code, noting that we are using pokemon_cat which has already casted certain variables as factors. What do you see?

ggplot(data = pokemon_cat, mapping = aes(x = leg_status)) +
  geom_bar() + 
  labs(x = "Count", y = "Legendary Status") +
  theme(text=element_text(size=20))

fct_infreq

By default, the bars are ordered alphabetically. One common data visualization task is to order our plot by frequency. We can easily do this using the fct_infreq() function. Examine the code used to create the following plot:

ggplot(data = pokemon_cat, mapping = aes(x = fct_infreq(leg_status))) +
  geom_bar() + 
  labs(x = "Count", y = "Legendary Status") +
  theme(text=element_text(size=20))

fct_relevel

We can also manually reorder the levels using the fct_relevel() function. For instance, let’s say we wanted to relevel the factor levels in order of decreasing “specialness.” We would use the following code:

pokemon_cat <- pokemon_cat %>%
 mutate(leg_status = fct_relevel(leg_status, 
                                 "Mythical", 
                                 "Legendary",
                                 "Sub Legendary",
                                 "Not Legendary"))


ggplot(data = pokemon_cat, mapping = aes(x = leg_status)) +
  geom_bar() +
  labs(x = "Count", y = "Legendary Status") +
  theme(text=element_text(size=20))

Note that the new plot used the releveled factor as the default order.

fct_recode

Another common task is to manually recode factor levels. That is, change existing factor levels to new labels. For instance, let’s suppose we wanted to recode the leg_status variable such that legendary and mythical Pokemon are tagged as legendary, and non- or sub-legendary Pokemon are not. We may use the following code to do so:

pokemon_cat <- pokemon_cat %>% 
 mutate(binary_leg = fct_recode(leg_status,
                              "No" = "Not Legendary",
                              "No" = "Sub Legendary",
                              "Yes" = "Legendary",
                              "Yes" = "Mythical")) 

# Verify it works:
pokemon_cat %>% 
  count(binary_leg)

fct_collapse

Similarly, we can combine many factor levels into one by using the fct_collapse() function. Let’s do the same task as the previous example, this time using fct_collapse():

pokemon_cat <- pokemon_cat %>% 
 mutate(collapse_leg = fct_collapse(leg_status,
                              no = c("Not Legendary", "Sub Legendary"),
                              yes = c("Legendary", "Mythical")))

# Verify it works:
pokemon %>% 
  count(collapse_leg)

fct_lump

A final often-performed command is to lump less common categories together (perhaps in an “Other” category). fct_lump() allows us to do so by either defining the absolute number of categories or by looking at relative proportions of the whole. For instance, the following code would allow us to lump all Pokemon whose primary types (type_1) that aren’t in the top 5 most common as “Other” (and create a new variable called prim_type along the way):

pokemon_cat %>% 
 mutate(prim_type = fct_lump(type_1, n = 5))

Similarly, the following code allows us to lump any Pokemon whose primary type (type_1) doesn’t comprise at least 5% of the whole into a separate “Other” category. Once again, we’re creating a new variable, prim_type:

pokemon_cat %>% 
 mutate(prim_type = fct_lump(type_1, prop = 0.05))
