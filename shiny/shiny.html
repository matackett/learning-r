Shiny is an R package that makes it easy to build interactive web apps straight from R. You can host standalone apps on a webpage or embed them in R Markdown documents or build dashboards. You can also extend your Shiny apps with CSS themes, htmlwidgets, and JavaScript actions. To get an idea of Shiny’s capabilities check our the Shiny gallery at https://shiny.rstudio.com/gallery/.

This tutorial serves as a gentle introduction to Shiny and covers the essentials in Shiny’s Part 1 - How to build a Shiny app video. For advanced Shiny topics and concepts, check out Shiny’s own tutorial and resources at https://shiny.rstudio.com/tutorial/.

Getting started

Instead of coding in an R Markdown file, you will develop your Shiny app in a script, .R, file. To get started

  1. open RStudio;

  2. if not installed, run install.packages("shiny");

  3. go to File > New File > Shiny Web App;

  4. enter your application’s name;

  5. keep option Single File (app.R) selected;

  6. enter the directory of where the application should be saved;

  7. a file app.R should open. This is a template app that serves as a good starting point.

The template app uses a dataset named faithful that is available in the base package of R. This dataset contains waiting times between eruptions and the duration of the eruption for the Old Faithful geyser in Yellowstone National Park, Wyoming, USA. Variables eruptions and waiting are both in minutes. This tutorial will build on the template app as a way to introduce key components of Shiny.

Click the Run App button. Your template app should look similar to what you see below.

App

Old Faithful Geyser Data

Code

# Define UI for application that draws a histogram
ui <- fluidPage(

    # Application title
    titlePanel("Old Faithful Geyser Data"),

    # Sidebar with a slider input for number of bins 
    sidebarLayout(
        sidebarPanel(
            sliderInput("bins",
                        "Number of bins:",
                        min = 1,
                        max = 50,
                        value = 30)
        ),

        # Show a plot of the generated distribution
        mainPanel(
           plotOutput("distPlot")
        )
    )
)

# Define server logic required to draw a histogram
server <- function(input, output) {

    output$distPlot <- renderPlot({
        # generate bins based on input$bins from ui.R
        x    <- faithful[, 2]
        bins <- seq(min(x), max(x), length.out = input$bins + 1)

        # draw the histogram with the specified number of bins
        hist(x, breaks = bins, col = 'darkgray', border = 'white')
    })
}

# Run the application 
shinyApp(ui = ui, server = server)

Shiny framework

A Shiny app is comprised of two main parts: a user interface (ui) and a set of server instructions. These exist together in a single file named app.R.

User interface (UI)

The user interface of a Shiny app controls the layout, positioning of items, and overall appearance of your app. This is what users see when they interact with your app. Generally, the UI will consist of a collection of input widgets and output displays. In the Old Faithful Geyser Data app from last section, there was one input widget (slider input) and one output display (plot). Shiny apps can also support action buttons, multiple tabs or pages, themes, and much more.

The user interface is built with HTML. The package shiny has helpful wrapper functions that facilitate app development without a deep understanding of HTML. For example, the slider widget from the Old Faithful Geyser Data app was created with the R function sliderInput(). Run the code below to see how this creates a slider widget. The corresponding HTML is given further below.

sliderInput(inputId = "bins", label = "Number of bins:", min = 1, max = 50, value = 30)

Rather than having to master a lot of fancy HTML, you only need to know the R functions available to create specific widgets. shiny has structured these functions in such a way that is makes it relatively easy to build a fancy UI.

Server

Every Shiny app is maintained by a computer running R. This could be your own computer (see Figure 1) or a remote server (see Figure 2). When you develop your app, your computer will run a session of R. When you go to publish your app, a remote server will serve as the computer running the R session.



The server provides a set of R instructions for how the app should react to user actions. These instructions live inside the server() function of the Shiny app framework. As you change the value of the slider widget in the template app, a set of R code in the server function takes that value and replots the histogram.

Old Faithful Geyser Data

The R server code uses the input Id name of the input widgets as way to reference the current value of the widget. In this example, the value for the number of bins for the histogram is connected through the input Id of "bins".

Available widgets

Check out the Shiny Widgets Gallery to see some of the available widgets. Each of these has two common arguments: inputId and label. The inputID takes a character value that will be used internally to access the value of the widget. The label is the displayed value for the widget in the user interface. It is what users will see when they interact with your app. All other arguments are widget-specific. Always look at the help when working with a new widget.

Display reactive output

Prerequisites

Update the user interface to include space for a data table to be output below the histogram. Use function dataTableOutput() inside function mainPanel() and below plotOutput(). All you need to provide in dataTableOutput() is an outputId value that will be used in the server function. Separate functions plotOutput() and dataTableOutput() with a comma.

mainPanel(
   plotOutput(outputId = "distPlot"),
   dataTableOutput(outputId = "datatable")
)

After you make the changes, run your app. You should see no changes in the user interface. All you have done is allocate space in your user interface for a data table to be displayed.

Server function

Including plotOutput() and dataTableOutput() in the user interface tells Shiny where to position these objects. To tell Shiny how to create these objects you’ll need to provide R code in the server function.

The server function plays a special role in the Shiny process; it builds a list-like object named output that contains all of the code needed to update the R objects in your app. Each R object needs to have its own entry in the list. Your template app currently has one entry, output$distPlot.

Each output entry is connected with a render*() function. These render functions allow you to incorporate the reactive values from widgets into your R code. There is a connection between the render function you use and the output you want to display.

Output function in UI Creates
dataTableOutput DataTable
htmlOutput raw HTML
imageOutput image
plotOutput plot
tableOutput table
textOutput text
uiOutput raw HTML
verbatimTextOutput text
Render function in server Creates
renderDataTable DataTable
renderImage images (saved as a link to a source file)
renderPlot plots
renderPrint any printed output
renderTable data frame, matrix, other table like structures
renderText character strings
renderUI a Shiny tag object or HTML

Use widget values

In the server function, input is another list-like object. It holds all the values of the widgets in your app. To access these values inside a render function, use the form input$<id_for_widget>; where <id_for_widget> is replaced with the widget Id you used in your UI. For example, to reference the slider input’s value in a render function, use input$bins. This is how you create reactivity in your Shiny app and connect the inputs to the outputs.

Note! Keep in mind that these reactive values can only be used in reactive style functions within your server function. The render functions are one set of functions that support reactive values.

server <- function(input, output) {

  output$distPlot <- renderPlot({
    faithful %>% 
      ggplot(aes(x = waiting)) +
      geom_histogram(bins = input$bins, fill = input$fill_color,
