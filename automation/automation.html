Introduction

Learning objectives

  • When to use iteration
  • Understand the for loop structure

Writing and using functions is one way to reduce duplication in your code. However, functions alone will not be able to eliminate all instances of duplication. You may be able to write a function to scale each numeric variable in a data frame to a [0, 1] scale. But, if you need to apply this function to 30 variables in that data frame there will be duplication. This is where iteration (automation) plays a role in reducing duplication. R offers loops and functionals as tools for reducing duplication through iteration. This tutorial will introduce you to automation through for loops.

Note! A functional is a function that takes a function as its argument and returns a vector as output.

Why automation?

Consider the small tibble below.

x <- tibble(
  col_a = c(3, -1, 0, 10),
  col_b = c(2, -2, 2, -2),
  col_c = c(8, sqrt(131), log(4), 33),
  col_d = 1:4
)

x

Suppose we were interested in computing the mean of each column. One technique may be as follows.

x %>% 
  pull(1) %>% 
  mean()
x %>% 
  pull(2) %>% 
  mean()
x %>% 
  pull(3) %>% 
  mean()
x %>% 
  pull(4) %>% 
  mean()

Unfortunately, this results in 12 lines of code with a lot of duplication. A slightly better technique would be to first create a function and then use that function.

get_column_mean <- function(data, index) {
  data %>% 
    pull(index) %>% 
    mean()
}

To get the column means of x, we can now use the following.

get_column_mean(data = x, index = 1)
get_column_mean(data = x, index = 2)
get_column_mean(data = x, index = 3)
get_column_mean(data = x, index = 4)

We’ve reduced the number of lines of code from 12 to nine. However, there still exists four lines of nearly the same code. All that varies is the column index we use to compute the mean. This works okay here since we only have four columns in our data frame. What if we had 100, or 1,000, or even 10,000 columns? Data this large is not uncommon, and we certainly do not want to have 10,000 function calls of get_column_mean() where the only variation is in the index.

Iteration with for loops

Looking back at our previous code, we see that the only variation is with regards to the column index in which we want to compute the mean. A for loop is one type of loop available in R that can easily automate our process and eliminate this duplication.

get_column_mean(data = x, index = 1)
get_column_mean(data = x, index = 2)
get_column_mean(data = x, index = 3)
get_column_mean(data = x, index = 4)

Note! R also has while and repeat loops. The for loop is the most common and is available in almost every programming language.

Syntax

Let’s simplify our repeated code by using a for loop. Below we’ll discuss the key components. Run the code below and verify the means match what we saw in the last section.

output <- numeric(ncol(x))  # output vector to store results, line 1

for (i in seq_along(x)) {   # define our looping sequence, line 3
  output[i] <- x  %>%       # code to iterate, lines 4 - 6
    pull(i) %>% 
    mean()
}

output

for loop components

  1. Before the loop, you’ll need to create an object to store your results. In the example above, we created a numeric vector that is the same length as the number of columns in our data frame x.

    output <- numeric(ncol(x))
output

    By default, each element of the vector is 0. Soon we’ll fill these in with our mean calculations.

  2. Next you’ll need to define the looping sequence. This begins with the R keyword for and is wrapped in parentheses. The variable i is a dummy variable. Let’s look at how seq_along() works with our data frame x.

    seq_along(x)

    This is a convenient way to get a vector from 1 to the number of columns in a data frame. The sequence, i in seq_along(x) means that each run of our loop, i will take on the next value in seq_along(x), starting with 1 the first time through the loop.

  3. The last main component is the loop’s body. This is code we have seen before. The code is run repeatedly each time for a different value of i. The first time through we would have

    output[1] <- x %>% 
 pull(1) %>% 
 mean()

output

    The second time through the loop we would have

    output[2] <- x %>% 
 pull(2) %>% 
 mean()

output

    The iteration stops once i reaches the end of the looping sequence. In our small example, that would be once the code body is run for when i is assigned the value of 4.

Iteration caution

Many of the functions available in base R and other packages are vectorized. This means that if you have a vector x with n elements and pass this vector into a vectorized function, the function will be applied to each element of x. For example:

x <- -4:4
x

abs(x)
x <- -4:4
x

sqrt(abs(x))

Although you could, using a for loop to do the above computations is not recommended. It is inefficient, and you will not be harnessing R’s full power and functionality. Run the code below to verify you get the same answer as above.

x <- -4:4
output <- numeric(length(x))

for (k in 1:length(x)) {
  output[k] <- sqrt(abs(x[k]))
}

output

To determine if you’ll need a loop to tackle your task or not, first look at the help for the functions you plan to work with to understand if they are vectorized or not. Most functions you apply to a flat vector will be vectorized and a loop will not be required. If you need to apply a function to each column of a data frame or each component in a list, then you may need to accomplish that task with a for loop.
